# エンジニアリングルール

## 成果物ごとの役割

- コードは **How** を表現する
- テストは **What** を表現する
- コミットメッセージは **Why** を表現する
- コードコメントは **Why not** を表現する

## ADRの遵守

- 作業を開始する前に、関連する `$HOME/.codex/docs/adr/` を特定して確認すること。
- 実装は `$HOME/.codex/docs/adr/` の決定事項に厳密に従うこと。

## コメント

- 適切なコメントを記述すること。
- コメントがなくても理解しやすいコードを目指しつつ、それ以外の方法では表現できない部分にはコメントを含めること。

## Skills

作業内容に関連する Skill が `$HOME/.agents/skills/` に存在する場合は、作業を開始する前に該当する `SKILL.md` を読み、その指示に従うこと。

複数の Skill が該当する場合は、必要なものをすべて適用すること。

以下の Skills は必ず読みなさい。

- git-rules
